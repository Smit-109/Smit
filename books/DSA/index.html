Fundamentals

Core concepts every programmer needs: how algorithms work, how to measure their performance, essential math, and famous computational puzzles.

Algorithm Basics

Definition

What is an Algorithm?

An algorithm is a finite sequence of unambiguous instructions that transforms some input into a desired output. Think of it as a recipe: you start with ingredients (input), follow steps, and arrive at a finished dish (output).

For instance, to check if a number is even, one algorithm could be: verify whether the number leaves no remainder when divided by 2.

Definition

Iteration vs Recursion

Iterative algorithms repeat a block of code using loops (for, while). They process elements one after another until a condition is met.

Recursive algorithms solve a problem by having a function call itself. Every recursive solution needs two things:

  • Base case — the simplest input where the answer is known directly.
  • Recursive step — expresses the current answer in terms of smaller sub-problems.

A classic example: computing x^n without a power operator. The insight is that x^n = x^(n/2) * x^(n/2) when n is even, and x^n = x * x^((n-1)/2) * x^((n-1)/2) when n is odd, with the base case x^0 = 1.

Key Concept

Call Stack & Stack Overflow

Each recursive call occupies a slot in the call stack. If recursion goes too deep, it can exceed the maximum stack size, causing a stack overflow. Converting recursion to iteration avoids this because heap memory is typically much larger than stack space.

Technique

Memoization

Cache the result of expensive function calls. When the same inputs appear again, return the cached value instantly. This transforms many exponential-time recursive algorithms into polynomial-time solutions.

Algorithm Paradigms

Brute Force

Enumerate every possible answer and test each one. Simple but often the slowest approach.

Example: Finding a pair summing to S in an array by trying all n^2 pairs.

Backtracking

Like brute force, but abandon invalid partial solutions early (pruning). Avoids wasting time on dead ends.

Example: N-Queens — place queens one at a time and backtrack whenever two can attack.

Greedy

Make the locally optimal choice at every step. Fast, but may not produce the globally optimal result.

Example: Picking the heaviest edge at each step — might miss a better route hidden behind a lighter edge.

Divide & Conquer

Split the problem into independent sub-problems, solve each, then combine. Merge sort and quick sort follow this strategy.

Dynamic Programming

Solve overlapping sub-problems once and reuse the results. Two styles:

  • Top-down — recursive + memoization
  • Bottom-up — iteratively fill a table from known base values

Example: Fibonacci numbers.

Key Distinction

D&C vs DP

Divide & Conquer works with independent sub-problems; DP works with overlapping ones. If you re-compute the same thing in a D&C solution, switch to DP.

Classification

P vs NP

  • P — problems solvable in polynomial time.
  • NP — solutions can be verified in polynomial time.
  • NP-hard — at least as hard as any NP problem.
  • NP-complete — NP-hard problems that are also in NP.

Whether P = NP remains one of the biggest open questions in computer science.

Pitfall

Integer Overflow

An N-bit integer cannot exceed 2^N - 1. To safely average a and b, prefer a + (b-a)/2 over (a+b)/2 — the latter can overflow before the division.

Classic Challenges

Traveling Salesperson Problem (TSP)

▶

Goal: Find the shortest tour visiting every city exactly once, returning to start.

Naive: try all (n-1)! permutations. Held-Karp (DP): For each subset of cities and possible last city, store minimum distance. O(n^2 2^n) O(n 2^n)

0/1 Knapsack Problem

▶

Goal: Choose items to maximize value without exceeding weight capacity C.

DP: Build table V[i][j] = max value using items 1..i with capacity j. Transition: V[i][j] = max(V[i-1][j], V[i-1][j-w_i] + v_i). Answer: V[n][C]. O(nC) O(nC)

N-Queens

▶

Goal: Place N queens on NxN board so no two attack each other.

Backtracking: Place queens one column at a time. Undo if invalid, try next row.

Coin Change

▶

Goal: Given unlimited coins of denominations {c1, ..., ck}, find min coins to make amount S.

Bottom-up DP: D[s] = min(D[s - c_i] + 1) over valid denominations. Answer: D[S]. O(S) O(S)

Data Structures

The building blocks for organizing and accessing data efficiently: arrays, strings, stacks, queues, hash-based structures, and linked lists.

Arrays & Strings

Definition

Array

A fixed-size container storing elements sequentially, accessed by index (0-indexed in most languages).

OperationTimeNotes
Access by indexO(1)Direct lookup
SearchO(n)Must scan until found
InsertionO(n)Shift elements; may resize
DeletionO(n)Shift elements left
Algorithm

Kadane's Algorithm (Maximum Subarray Sum)

Finds the contiguous subarray with the largest sum in a single pass:

  1. Initialize current = max = A[0].
  2. For each A[i]: if current < 0, restart from A[i]; otherwise, extend with A[i].
  3. Update max = max(max, current) at each step.
O(n) O(1)
Algorithm

Merge Intervals

Given potentially overlapping intervals, produce non-overlapping ones:

  1. Sort intervals by start value.
  2. Iterate: if current overlaps last merged, extend; otherwise append.
O(n log n) O(n)
Definition

String

Essentially an array of characters. Special types:

  • Palindrome — reads the same forwards and backwards (e.g., "racecar").
  • Anagram — rearrangement of another string's letters (e.g., "listen" / "silent").
  • Strobogrammatic — looks the same when rotated 180 degrees (e.g., "69", "818").
Technique

Sliding Window (Longest Substring Without Repeats)

Use two pointers (left, right) defining a window. Expand right; if a duplicate appears, shrink left until valid. Track max window length. O(n) O(n)

Stacks & Queues

Stack (LIFO)

Last In, First Out. Add and remove from the top.

  • push — O(1)
  • pop — O(1)
  • Access/search other positions: O(n)

Used in: DFS, undo operations.

Queue (FIFO)

First In, First Out. Add at tail, remove from head.

  • enqueue — O(1)
  • dequeue — O(1)
  • Access/search other positions: O(n)

Used in: BFS, task scheduling.

Application

Monotonic Stack (Daily Temperatures)

Given daily temperatures, find days until a warmer one. Use a stack of (day, temp) pairs. When a new temp arrives, pop all cooler days and record the gap. O(n) O(n)

Hash Tables

Definition

Hash Set & Hash Map

A hash set stores unique values; a hash map stores key-value pairs. Both use a hash function to map keys to bucket addresses for near-O(1) operations. A good hash function should: (1) be fast, (2) distribute uniformly, (3) handle collisions.

Mechanism

Collision Resolution

A collision occurs when two distinct keys hash to the same bucket. Load factor ρ = n/b (items/buckets).

MethodTypeHow it worksBest when
Linear probingOpen addressingTry next bucket sequentiallyρ ≈ 0
Quadratic probingOpen addressingTry buckets at 1,4,9,... awayρ ≈ 0
Double hashingOpen addressingUse a second hash functionρ ≈ 0
ChainingClosed addressingEach bucket holds a linked listρ ≈ 1

Probabilistic Structures

Definition

Bloom Filter

Space-efficient structure that tells if an element is definitely not in a set or possibly in it (false positives possible, never false negatives). Uses k hash functions and an m-bit array. Cannot delete. False positive rate approx (1 - e^{-kn/m})^k.

Definition

Count-Min Sketch

Estimates element frequencies (returns upper bound). Uses k hash functions and a k×m count matrix. Insert x: increment cells. Query x: return min across all k counters. Cannot delete.

Linked Lists

Singly Linked List

Each node has value and next pointer. First = head.

OpTime
Access / SearchO(n)
Insert / Delete (known pos)O(1)
Doubly Linked List

Each node also has prev pointer. First = head, last = tail.

OpTime
Access / SearchO(n)
Insert / Delete (known pos)O(1)
Algorithm

Floyd's Cycle Detection (Tortoise & Hare)

Detects cycle start using two pointers:

  1. Meeting: Slow moves 1 step, fast moves 2. They meet inside the cycle.
  2. Reset: Move slow back to head; keep fast at meeting point.
  3. Detect: Move both 1 step at a time. They meet at cycle start.
O(n) O(1)
Application

LRU Cache

Combines a doubly linked list (most recent at head, least at tail) with a hash map (key to node) for O(1) access. Access: find and move to head. Insert new: evict tail if at capacity, add at head. Insert existing: move to head. O(1) O(C)

Graphs & Trees

Graph traversal, shortest paths, spanning trees, and tree structures from basic binary trees to advanced self-balancing and range-query trees.

Graph Fundamentals

Definition

Graph

A graph G = (V, E) consists of vertices and edges. Undirected edges have no direction; directed edges go one way. Represented by adjacency list (O(V+E) space) or adjacency matrix (O(V²) space).

Properties

Degree & Graph Types

  • Degree = edges touching a node. Directed: in-degree, out-degree.
  • Handshaking Lemma: Sum of degrees = 2|E|. Odd-degree nodes come in even count.
  • DAG = Directed Acyclic Graph.
  • Types: Complete (all pairs connected), Connected (path between every pair), Bipartite (two disjoint sets with edges only crossing between them).

Graph Traversal

BFS

Level by level, using a queue. O(V+E) O(V)

  1. Enqueue start node.
  2. Dequeue, mark visited, enqueue unvisited neighbors.
  3. Repeat until empty.
DFS

Go deep first, using a stack or recursion. O(V+E) O(V)

  1. Push start node.
  2. Pop, mark visited, push unvisited neighbors.
  3. Repeat until empty.
Algorithm

Topological Sort (Kahn's Algorithm)

Linear ordering of a DAG where every edge (A,B) places A before B:

  1. Compute in-degree of every node.
  2. Enqueue nodes with in-degree 0.
  3. Dequeue, add to result, decrement neighbors. Enqueue any reaching 0.
  4. Repeat. If result lacks all nodes, a cycle exists.
O(V+E) O(V)

Dijkstra's Algorithm

▶

Greedy algorithm for non-negative weighted graphs:

  1. Set source distance = 0; all others = infinity.
  2. Pick unvisited node with smallest distance. Visit it.
  3. Update neighbors' distances if path through current node is shorter.
  4. Repeat until all visited.

No negative edges: Once visited, distance is finalized.

A* Search

▶

Extends Dijkstra's with a heuristic h(j, target) estimating remaining distance. Node selection: d(source,j) + h(j,target). Biases exploration toward the goal.

Bellman-Ford

▶

Relaxes every edge up to V-1 times. Supports negative weights but not negative cycles. If distances still decrease after V-1 rounds, a negative cycle exists.

Floyd-Warshall

▶

Computes all-pairs shortest paths. For each intermediary k, check if i→k→j is shorter than current i→j. O(V³) O(V²)

Spanning Trees & Components

Definition

Spanning Tree & MST

A spanning tree is a subgraph with the minimum edges to keep all vertices connected. A minimum spanning tree (MST) minimizes total edge weight. A graph can have multiple MSTs.

Prim's Algorithm

Grow MST from an arbitrary node by always adding the cheapest edge connecting a visited node to an unvisited one. O(E log V)

Kruskal's Algorithm

Sort all edges by weight. Add each edge if it doesn't create a cycle (use union-find). O(E log E)

Algorithm

Kosaraju's Algorithm (Strongly Connected Components)

Finds SCCs in a directed graph in two passes:

  1. First pass: DFS on original graph. Push each node to a stack after all neighbors are visited.
  2. Second pass: DFS on reversed graph, processing nodes in stack order. Each DFS tree is one SCC.
O(V+E) O(V)

Definition

Heap

A complete binary tree where each node satisfies the min-heap (parent ≤ children) or max-heap (parent ≥ children) property. Since it's complete, it can be stored in an array: parent of i is (i-1)/2, children are 2i+1 and 2i+2.

Definition

Binary Search Tree (BST)

A binary tree where each node's value is greater than all values in its left subtree and less than all values in its right subtree. Height ranges from O(log n) when balanced to O(n) when degenerate.

Definition

Trie (Prefix Tree)

An N-ary tree for efficient word storage and retrieval. The path from root to any node spells a prefix. Each node has a character, children hash map, and a boolean flag indicating if the path forms a complete word. Search, insert, and delete all run in O(m) where m = word length.

Advanced Trees

Definition

Self-Balancing Trees

Trees that maintain O(log n) height through rotations:

Mechanism

Tree Rotations

Left rotation: right child becomes new root, original root becomes left child of new root. Right rotation: mirror image. Both preserve in-order traversal and take O(1) time.

Prefix Sum: P[i] = A[0]+...+A[i]. Range sum [i,j] = P[j] - P[i-1].

Fenwick Tree: Uses binary representation of indices to define responsibility ranges. Efficient for point updates and prefix sum queries.

Segment Tree: Binary tree where each node covers a range. Leaf = single element; internal node = aggregate of children. Supports min, max, sum queries.

Sorting & Searching

The most common sorting algorithms, from simple quadratic methods to efficient O(n log n) approaches, plus essential search techniques including string pattern matching.

Bubble Sort

▶

Compare consecutive elements and swap if out of order. Repeat until no swaps needed. After the k-th pass, the last k elements are in their final positions. Best case (already sorted): 1 pass.

Insertion Sort

▶

Build a sorted subarray one element at a time. For each new element, shift larger elements right and insert it at the correct position. Efficient for small or nearly-sorted arrays.

Selection Sort

▶

Find the minimum unsorted element and swap it to the front. Repeat for the remaining portion. Always makes O(n²) comparisons regardless of input.

Efficient Sorts

AlgorithmBestAverageWorstSpaceStable
Merge SortO(n log n)O(n log n)O(n log n)O(n)Yes
Heap SortO(n log n)O(n log n)O(n log n)O(1)No

Merge Sort

▶

Divide the array into single-element subarrays, then merge pairs using the two-pointer technique until one sorted array remains. Usually implemented recursively. Guarantees O(n log n) in all cases.

Heap Sort

▶

Build a max-heap from the array. Repeatedly extract the maximum (swap root with last element, heapify down) to build the sorted array from the end. In-place and O(n log n) guaranteed.

Quick Sort

▶

Choose a pivot, partition the array so smaller elements go left and larger go right, then recurse. Worst case O(n²) when pivot is extreme; average O(n log n). Pivot strategies: random choice, or median-of-three. Not stable but fast in practice.

Note

Stability

A sorting algorithm is stable if equal elements maintain their original relative order. Merge sort and insertion sort are stable; heap sort and quick sort are not.

Specialized Sorts

Algorithm

Counting Sort

Count occurrences of each value, compute cumulative counts, then place each element at its correct position. Stable and linear when k = O(n).

Algorithm

Radix Sort

Sort elements digit by digit using counting sort, starting from the least significant digit. The stability of counting sort ensures that earlier (weaker) digit orderings are preserved when later (stronger) digits are processed.

Search Techniques

Technique

Linear Search

Scan elements one by one until the target is found or the end is reached. O(n) O(1)

Technique

Two-Pointer Technique

Use two pointers (often at opposite ends) to scan a sorted array efficiently. Example: find a pair summing to S by adjusting pointers based on the current sum vs. target. O(n)

Algorithm

Binary Search

Find a target in a sorted array by repeatedly halving the search space:

  1. Set left = 0, right = n-1.
  2. Compute mid = left + (right-left)/2 (avoids overflow).
  3. If A[mid] equals target: found. If A[mid] < target: search right half. If A[mid] > target: search left half.
  4. Repeat until found or left > right (not found).
O(log n) O(1)
Application

Trapping Rain Water

Given building heights, find trapped water volume. Water at index i = min(tallest left, tallest right) - height[i]. Use two pointers from both ends, tracking max heights. O(n) O(1)

Pattern Matching

Definition

String Pattern Matching

Given a string s (length n) and a pattern p (length m), determine if p appears in s.

Algorithm

Brute-Force Approach

Check every possible starting position in s against p. O(n×m) O(1)

Algorithm

Knuth-Morris-Pratt (KMP)

Avoid restarting the pattern search from scratch after a failed match. Build a prefix-suffix array that indicates where to resume matching:

  1. Build prefix-suffix array: For each position i in the pattern, compute the length of the longest proper prefix that is also a suffix of p[0..i]. O(m)
  2. Search: When a mismatch occurs at pattern position j, jump to position q[j-1] instead of restarting from 0. O(n)
Total: O(n+m) O(m)
Algorithm

Rabin-Karp

Use hashing to compare the pattern against substrings of the same length. Compute a rolling hash so each new substring hash can be derived from the previous one in O(1) time. If hashes match, verify character-by-character to rule out false positives. Expected O(n+m) but worst case O(n×m) with many collisions.
