5.2 Classification Tasks

5.2.1 Sentiment Extraction

Task

Predict whether input text has positive or negative sentiment

Datasets

  • Product reviews (Amazon, Yelp)
  • Movie reviews (IMDB)
  • Social media (Reddit, Twitter)

Metrics

  • Accuracy: Overall performance
  • Precision: Accuracy of positive predictions
  • Recall: Coverage of actual positives
  • F1: Harmonic mean of precision/recall

5.2.2 Named Entity Recognition (NER)

Task

Predict token categories (entities): PERSON, LOCATION, ORGANIZATION, etc.

Dataset

CoNLL-2003: annotated text with PER, LOC, ORG, MISC

Metrics

  • Per-class precision, recall, F1
  • Macro-averaged across classes
Macro-Metric = (1/|C|) Σc∈C Metricc

5.3 Generation Tasks

5.3.1 Machine Translation

Task

Translate from source to target language while maintaining meaning and fluency

Datasets

DatasetSizeDescription
WMTMillionsVarious sources, news
EuroparlMillionsEuropean Parliament
IWSLT~MillionsTED talks, spoken

Language Categories

  • High-resource: Plenty of data (English, French, German)
  • Low-resource: Limited data (dialects, rare languages)

Example: NLLB covers 200+ languages including low-resource

Perplexity

PP(X) = exp(-(1/n) Σi=1n log P(xi|x1,...,xi-1))
  • PP → 1: Model is confident
  • PP → ∞: Model is uncertain

BLEU Score

BiLingual Evaluation Understudy - measures n-gram precision:

BLEU = BP · exp((1/N) Σn=1N log pn)

Where:

  • pn = n-gram precision
  • BP = Brevity Penalty (prevents short translations)
BP = min(1, exp(1 - nref/npred))

METEOR

Metric for Evaluation of Translation with Explicit Ordering

METEOR = Fmean · (1 - p)

Uses precision, recall, and chunk penalty for word order

WER

Word Error Rate for speech recognition:

WER = (nS + nD + nI) / n
  • nS: Substitutions
  • nD: Deletions
  • nI: Insertions

ROUGE

Recall-Oriented Understudy for Gisting Evaluation

ROUGE-N

N-gram overlap:

ROUGE-N Precision = matched N-grams / predicted N-grams
ROUGE-N Recall = matched N-grams / reference N-grams
ROUGE-L

Longest Common Subsequence:

ROUGE-L F1 = (2 · LCS / prediction + reference)
ROUGE-S

Skip bigrams (non-contiguous pairs)

Common ROUGE Metrics

F1 scores of ROUGE-1, ROUGE-2, ROUGE-L

5.3.3 Retrieval-Augmented Generation (RAG)

Motivation

  • Training is expensive and done once
  • Knowledge can be outdated (cutoff date)
  • Hallucination risk

Solution

Combine LLM knowledge with external documents

User Question
      ↓
[Retriever] ←→ [Vector Database]
      ↓
[Augmented Prompt]
      ↓
[Generator (LLM)]
      ↓
Answer

Retriever

  1. Embed prompt: q = Eq(query)
  2. Search: Compute similarity with documents
  3. Select: Top-K most relevant documents

Generator

  1. Augment: Add retrieved docs to context
  2. Generate: Produce response with augmented prompt

Training

  • Retriever: Train Ed for document embeddings
  • Generator: Train LLM with augmented prompts

RAG vs Vanilla LLM

AspectVanilla LLMRAG
KnowledgeTraining data onlyTraining + external
HallucinationHigher riskLower risk
FreshnessLimited by cutoffUpdated via sources
