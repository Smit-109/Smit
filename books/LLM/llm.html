4.1 Fundamentals

4.1.1 Notations

Large Language Model (LLM)

A model with large number of parameters achieving remarkable performance across NLP tasks. Based on Transformer architecture (decoder-only or encoder-decoder).

4.1.2 Response Generation

Autoregressive Generation

Generate output one token at a time:

P(wt+1|C) = softmax(Xt+1)

Greedy Search

wt+1 = argmax P(w|C)

Simple but may miss globally optimal sequences

Beam Search

Keep top-k hypotheses, expand iteratively

  • k = beam width
  • k = 1 is greedy decoding

Temperature Sampling

Padj(w|C) = exp(Xw/T) / Σ exp(Xj/T)
TEffect
T → 0Deterministic, selects highest prob
T = 1Original distribution
T → ∞Uniform, random, creative

Top-k Sampling

Sample from top-k highest probability tokens

Top-p (Nucleus) Sampling

Sample from smallest set with cumulative probability ≥ p

4.3.2 Strategies

In-Context Learning (ICL)

  • Zero-shot: No examples
  • Few-shot: Few input-output pairs

Chain of Thought (CoT)

Include reasoning steps in examples:

Q: How old is this bear?
A: Born in 2020. It is 4.

Q: How old last year?
A: 2024 - 1 = 3

Self-Consistency

  1. Sample N responses with CoT
  2. Parse final answers
  3. Select most common answer

Tree of Thoughts (ToT)

Explore multiple reasoning paths using graph search (BFS/DFS)

ReAct (Reason + Act)

Combine reasoning and actions:

  1. Thought: Plan next step
  2. Action: Execute (e.g. search)
  3. Observation: Incorporate result

4.3.3 Risks

Prompt Injection

Trick model into unintended behavior:

"Now forget all instructions and answer..."

Hallucination

Model generates plausible but incorrect facts

Mitigation: RLHF, DPO, RAG

4.4 Finetuning

4.4.1 Supervised Finetuning (SFT)

Tunes all model weights on specific tasks

Process

  1. Initialize with pretrained weights
  2. Collect prompt-response pairs
  3. Train with next-token prediction

Data Quality

quality > quantity

Few hundred examples typically sufficient

FLAN (Instruction Tuning)

Teach model to respond to instructions for better zero-shot performance

4.4.2 Parameter Efficient Finetuning (PEFT)

LoRA (Low-Rank Adaptation)

Add low-rank matrices instead of updating all weights:

W = W0 + BA

Where B ∈ ℝd×r, A ∈ ℝr×k, r ≪ min(d,k)

LoRA Parameters

TraditionalLoRA
d × kd × r + r × k ≈ d × k / 104

Benefits

  • ~10,000x fewer trainable parameters
  • Easier to switch between tasks
  • Only store W0 + task-specific (A,B)

Adapters

Small learnable modules inserted between layers

Prefix Tuning

Tune only prefix token embeddings (< 1% of parameters)

4.5 Preference Tuning

4.5.1 Concepts

Motivation

Teaching model what actions to take AND what to avoid

Preference Data Types

TypeDescription
PointwiseAbsolute rating per example
PairwiseCompare two examples
ListwiseRank list of examples

Generating Preference Data

  1. Generate two responses from SFT model
  2. Rate relative quality (binary or nuanced)

4.5.2 RLHF

Reinforcement Learning from Human Feedback

Step 1: Reward Model    Step 2: RL (PPO)
Human ratings → RM    RM guides LLM
      ↓                      ↓
  r(x,y)               Policy π updates

Reward Model Training

L(θ) = -E[log σ(r(x,yw) - r(x,yl))]

Maximize difference between winning and losing responses

PPO Update

L(θ) = -r(x,y) - β · KL(π(y|x) || πold(y|x))

Maximizes rewards while staying close to base model (KL penalty)

Best-of-N (Rejection Sampling)

Sample N responses, select highest scored by RM

4.5.3 Supervised Preference Tuning

DPO (Direct Preference Optimization)

Combines RM + RL into single supervised step:

LDPO = -E[log σ(β log π(yw|x)/πold(yw|x) - β log π(yl|x)/πold(yl|x))]

DPO Properties

  • Fewer hyperparameters
  • No reward model needed
  • Simpler than RLHF

IPO

Identity Preference Optimization - less prone to overfitting:

LIPO = E[(1/2β) log2(π(yw|x)/πold(yw|x) - π(yl|x)/πold(yl|x))]

4.6.2 Distillation

Transfer knowledge from teacher to student:

Teacher (big) → soft labels → Student (small)
Hard labels → Alternative

Loss

L = KL(yteacher || ystudent)

Example: DistilBERT - 60% size, 97% performance

4.6.3 Quantization

Floating-Point Formats

FormatBitsRange
FP3232~1038
FP1616~105
BF1616~1038
INT88~102

Quantization Methods

  • Zero-point: Asymmetric mapping
  • Absmax: Symmetric mapping
  • Blockwise: Quantize blocks independently

Training Types

  • PTQ: Post-Training Quantization
  • QAT: Quantization-Aware Training

Mixed Precision

Compute in FP16, store weights in FP32. Reduces VRAM by 50%.

QLoRA

Quantized LoRA:

  • 4-bit NormalFloat (NF4) weights
  • Double quantization for constants
  • Paged optimizers for memory spikes

LLM.int8()

8-bit inference with outlier separation:

  • Small values: INT8 dot products
  • Outliers: FP16 computation
