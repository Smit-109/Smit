3.1 Self-Attention Mechanism

3.1.1 Query, Key, Value

Self-Attention

Allows each token to attend to all other tokens regardless of distance. Enables context-aware embeddings.

The Attention Mechanism

Three projections per token:

  • Query (Q): "What am I looking for?"
  • Key (K): "What do I contain?"
  • Value (V): "What information do I carry?"

Attention Formula

Attention(Q, K, V) = softmax(QKT / √dk)V

Where dk = dimension of keys (dq = dk)

Step-by-Step Process

  1. Compute dot products: q · ki for all tokens
  2. Scale by √dk
  3. Apply softmax to get weights pi
  4. Compute weighted sum: Σ pi · vi

Why Scale by √dk?

Variance of dot product increases with dimension. Scaling prevents softmax saturation.

Complexity

O(n2) where n = sequence length

3.1.2 Multi-Head Attention

h attention heads enable learning different relationship types in parallel:

Input Q,K,V
    ↓
[Head 1: W₁Q, W₁K, W₁V] → O₁
[Head 2: W₂Q, W₂K, W₂V] → O₂
    ...
[Head h: WhQ, WhK, WhV] → Oh
    ↓
Concat(O₁,...,Oₕ) · Wᴼ

Within Head i

Qi = XWiQ, Ki = XWiK, Vi = XWiV
Attention(Qi, Ki, Vi) = softmax(QiKiT/√dk)Vi

Parameters

NMHA = 2 · (dk + dv) · dmodel · h

Types of Attention

3.1.4 End-to-End Example (Translation)

Translation: "My teddy bear is cute." → "Mon ours en peluche est mignon."

Step 1: Tokenization

Convert source sentence to tokens using tokenizer T and vocabulary V

Step 2: Encoded Embeddings
  1. Token embedding: Look up dmodel-dim vector for each token
  2. Position embedding: Add position-specific vector
  3. Self-attention: Each token attends to all others
  4. Feed-forward: Project through FFNN
Step 3: Decoder Predictions
  1. Input [BOS] token
  2. Masked self-attention on decoded tokens
  3. Cross-attention with encoder output
  4. Project to vocabulary → softmax → next token
  5. Repeat until [EOS]

Training vs Inference

  • Training: Parallel (teacher forcing)
  • Inference: Autoregressive (one token at a time)

3.3 Computational Improvements

3.3.1 Position Awareness

Learned Embedding

  • Model learns position-specific embeddings
  • Cannot generalize beyond training length

Sinusoidal Embedding

PE(pos)2i = sin(pos / 100002i/dmodel)
PE(pos)2i+1 = cos(pos / 100002i/dmodel)
  • Hardcoded, no parameters
  • Generalizes beyond training length

RoPE (Rotary Position Embeddings)

Injects relative position via rotation matrices:

qm → qmRm, kn → knRn
  • Used by LLaMA
  • Flexible: no max length limitation

3.3.2 Layer-Related Tricks

Attention Variants

MethodDescription
MHAStandard multi-head attention
MQAMulti-Query: share K,V across heads (G=h)
GQAGrouped-Query: G groups (1<G<h)

GQA balances speed and quality. Used by LLaMA 2+.

Layer Normalization

μ = (1/d)Σxi, σ2 = (1/d)Σ(xi-μ)2
LN(x) = γ(x-μ)/√(σ2+ε) + β

Label Smoothing

Replaces hard labels with soft distribution:

labeli = (1-ε)·hardi + ε/|V|

Prevents overconfidence, improves generalization.

3.5 Attention Computation Speedup

3.5.1 Sparse Attention

Reformer

  • Uses LSH to group similar Q and K
  • Reduces complexity O(n2) → O(n log n)
  • Reversible layers for memory efficiency

Longformer

  • Sliding window attention for local context
  • Global attention for special tokens
  • Efficient for long documents

3.5.2 Low-Rank Attention

Linformer

attention(Q, K, V) ≈ softmax(Q(EPK)T/√dk) (EPV)V

Projects K, V to lower dimension k < n. Complexity O(n)

Performer

Uses kernel approximation (FAVOR+) for unbiased softmax estimation

3.5.3 Hardware Optimization

Flash Attention

  • Tiling: divide attention into blocks fitting in SRAM
  • Recomputation: don't store O(n2) activations
  • No approximation - exact results, faster execution

3.6 Interpretability

3.6.1 Attention-Specific Methods

3.6.2 Model-Agnostic Methods

TCAV

Testing with Concept Activation Vectors:

  1. Define concept from examples
  2. Train classifier on activations
  3. Compute directional derivative

Integrated Gradients

IGi = (xi - x̃i) · (1/m) Σk=1m ∂f(x̃ + (k/m)(x-x̃))/∂xi

LIME

Local Interpretable Model-agnostic Explanations

  1. Sample and perturb data
  2. Train interpretable model nearby
  3. Use coefficients for interpretation

TracIn

Traces influence of training examples on test predictions
