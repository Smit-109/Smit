Tokenization, Word Embeddings, and Document Representations

2.1 Tokenization

2.1.1 Notations

Token

An arbitrarily defined unit of text. Granularity levels:

LevelDescriptionProsCons
Word Each word is a token Easy to understand, fewer tokens Large vocabulary, OOV words
Subword Words divided into subunits Reduced vocab, handles OOV More complex, longer input
Character Each character is a token Small vocab, solves OOV Very long input, poor embeddings
Byte Raw encoding (e.g. UTF-8) Handles any text/emoji Long input, hard to interpret

Vocabulary

A fixed set of predefined tokens V = {token1, token2, ...}

Special Tokens

TokenPurposeExample

[UNK]Unknown tokens not in vocabmy teddy [UNK]

Tokenizer

Converts text to tokens:

  • Encode: Text → Tokens
  • Decode: Tokens → Text

Types

  • Rule-based: Word-level, character-level (no training)
  • Learned: BPE, WordPiece, Unigram (trained on corpus)

2.1.2 Subword Algorithms

BPE (Byte-Pair Encoding)

Constructs vocabulary by learning most common pairs:

Training:
  1. Initialize: Divide corpus into characters, count frequencies
  2. Count pairs: Count frequency of adjacent pairs
  3. Merge: Replace most common pair with new token
  4. Repeat until vocabulary reaches desired size
Encoding:
  1. Divide text into characters
  2. Apply merge rules in order

Used by: GPT, LLaMA

WordPiece

Similar to BPE but selects pairs based on maximum likelihood rather than frequency.

Used by: BERT

Unigram

Assumes token appearance is independent. Starts with large vocabulary and prunes:

  1. Initialize with all possible subwords
  2. Compute loss for each subword using EM algorithm
  3. Prune 80% with highest loss increase
  4. Repeat until desired vocabulary size

Used by: T5

2.2 Token Embeddings

2.2.1 One-Hot Encoding

Vector of size |V| with 1 at token index, 0 elsewhere:

tokeni → [0, ..., 0, 1, 0, ..., 0]

Limitations

  • Token similarity: All vectors orthogonal - no semantic similarity
  • Dimensionality: |V| can be 104-105

2.2.2 Continuous Encodings (Word2Vec)

Word2Vec

Generates continuous, fixed-dimension embeddings (typically dmodel ~ 102-103)

CBOW (Continuous Bag of Words)

Predicts target token from average of context token embeddings:

[Context] → Average → Projection → Prediction → [Target]
  my cute teddy       [Avg]       [Proj]       [Softmax]     bear

Fast training: One prediction per target token

Skip-gram

Predicts context tokens given target token:

[Target] → Projection → [Context 1, 2, ...]
 teddy          [Proj]       my cute is reading

Slower training: 2×C predictions per target token

Negative Sampling

Approximates softmax by sampling ~10 negative examples:

L = -log σ(wp · wt) - Σwn∈N log σ(-wn · wt)

GloVe

Global Vectors - uses co-occurrence matrix:

  1. Build co-occurrence matrix X
  2. Model: log(Xi,j) = wt,iTcj + bi + bj
  3. Learn embeddings via weighted loss
w = (wt + wc) / 2

2.3 Document Embeddings

2.3.1 Heuristic Methods

BOW (Bag of Words)

Sum of one-hot encodings weighted by frequency:

xBOW = [freq("this"), freq("is"), freq("a"), ...]

Limitation: Order doesn't matter

TF-IDF

Term Frequency × Inverse Document Frequency:

TF(t,d) = ft,d / Σt' ft',d
IDF(t,D) = log(ND / Nt,D)
TF-IDF(t,d,D) = TF(t,d) × IDF(t,D)

Filters out common words, focuses on important ones.

N-gram

BOW extended to sequences of n consecutive words. Trade-off:

  • Higher n → captures word colocation
  • Lower n → less sparse, lower computation

2.3.2 Recurrent Neural Networks

RNN

Maintains hidden state h(t) for sequence processing:

h(t) = A1(Whhh(t-1) + Whxx(t) + bh)
ŷ(t) = A2(Wyhh(t) + by)

Vanishing Gradient Problem

RNNs struggle with long-term dependencies. Gradients become small during backpropagation.

GRU (Gated Recurrent Unit)

Uses gates to control information flow:

  • Input gate i: Filter new information
  • Forget gate f: Discard information
c̃(t) = tanh(Wc[f ⊙ h(t-1), x(t)] + bc)
c(t) = i ⊙ c̃(t) + (1-i) ⊙ c(t-1)

LSTM

Adds output gate o for better long-term memory:

c(t) = f ⊙ c(t-1) + i ⊙ c̃(t)
h(t) = o ⊙ tanh(c(t))

ELMo

Embeddings from Language Models - bidirectional LSTMs with character-level input:

ELMot = [ht,L→, ht,L←]

2.3.3 Attention-Based Methods

Motivation

Attention injects context that's a direct function of past tokens, addressing vanishing gradients:

ci = Σj αi,j hj

Attention Weights

αi,j = exp(ei,j) / Σk exp(ei,k)

Where ei,j = score from alignment model A(hi-1, hj)

2.4.2 Dimension Reduction (t-SNE)

t-Distributed Stochastic Neighbor Embedding reduces D → d dimensions preserving similarities:

  1. Compute pairwise distances ||xi - xj|| in high-D space
  2. Convert to similarity scores using Gaussian kernel
  3. Initialize low-dim points yi
  4. Compute similarities in low-dim space using t-distribution
  5. Minimize KL-divergence via gradient descent

Perplexity: Controls balance between local and global aspects

2.4.3 Fast Retrieval (LSH)

Locality-Sensitive Hashing for efficient nearest neighbor search in O(log n):

  1. Generate K random hyperplanes
  2. Each hyperplane divides space into 0/1 regions
  3. Build hash table with K-bit codes
  4. Elements with same hash code → likely similar
  5. Verify within same bucket

Use cases: Real-time applications, approximate results, near-duplicate detection
