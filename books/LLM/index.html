Part 1

Neural Networks

Home › Part 1: Foundations › Neural Networks
⚙

Overview

A neural network is a computational system built from interconnected layers of simple processing units. Each unit (or neuron) receives numeric inputs, multiplies them by learnable weights, adds a bias term, and passes the result through a nonlinear activation function. Stacking these layers lets the model learn hierarchical representations—early layers capture simple patterns while deeper layers compose them into abstract concepts. For instance, when processing a weather forecast, initial layers might detect numeric trends, while deeper layers could learn that a pressure drop often signals incoming rain.

Components of a Neural Network

Input Layer — Receives the raw feature vector. If your data describes a recipe, the input layer ingests numeric representations of ingredients, quantities, and cooking times.
Hidden Layers — Perform successive transformations. Each hidden layer computes weighted sums followed by a nonlinearity, gradually reshaping the data into a more useful representation.
Weights & Biases — Every connection between neurons carries a weight, and each neuron has an additive bias. Together they encode what the network has learned from training data.
Activation Function — Introduces nonlinearity so the network can model complex relationships rather than being limited to linear maps.
Output Layer — Produces the final prediction or decision. Its size and activation depend on the task (e.g., softmax for classification, linear for regression).

Activation Functions

Function Formula Range Key Property
Sigmoid σ(x) = 1 / (1 + e-x) (0, 1) Saturates for large |x|, causing vanishing gradients; useful for binary gate outputs
Tanh tanh(x) = (ex - e-x) / (ex + e-x) (-1, 1) Zero-centered but still saturates; often preferred over sigmoid in hidden layers
ReLU f(x) = max(0, x) [0, ∞) Computationally cheap, mitigates vanishing gradients; suffers from "dying ReLU" problem
Leaky ReLU f(x) = x if x > 0, else αx (∞, ∞) Small negative slope α prevents neurons from dying; α is typically 0.01
GELU GELU(x) = x · Φ(x) Approx (-0.17, ∞) Smooth, probabilistic gating; standard in Transformer models like GPT and BERT
ELU f(x) = x if x > 0, else α(ex - 1) (−α, ∞) Pushes mean activations toward zero, speeding convergence; α controls saturation

Gated Units

🔒

Why Gating?

In sequences, information from earlier steps can be drowned out by later inputs. Gating mechanisms learn when to let information flow through and when to block it. Think of a robot assistant processing a multi-step command: it must remember the initial instruction ("fetch coffee") while ignoring irrelevant details in later steps ("and also walk around the table"). A gate acts like a learned traffic controller, deciding which parts of the previous state to keep or discard.

Output Layer

Choosing an Output Activation

For multi-class classification (e.g., categorizing a news article into sports, politics, or technology), apply softmax to produce a probability distribution. For binary decisions (e.g., is this email spam?), use sigmoid. For regression tasks (e.g., predicting tomorrow's temperature), use a linear output with no nonlinearity.

Part 1

Training

Home › Part 1: Foundations › Training

Backpropagation

🔄

Backpropagation is the algorithm that computes how much each weight contributed to the overall error. Starting from the output layer, it applies the chain rule layer by layer to propagate gradient signals backward through the network. Each weight receives a gradient indicating the direction and magnitude of change needed to reduce the loss. Without backpropagation, training deep networks would be impractical because manually computing gradients for millions of parameters is infeasible.

Parameter update: w ← w − η · ∂L/∂w

where η is the learning rate and ∂L/∂w is the gradient of the loss with respect to the weight.

Weight Initialization

Xavier (Glorot) Initialization

Sets weights drawn from a distribution with variance 2/(nin + nout), where nin and nout are the number of input and output units. Works well with sigmoid and tanh activations because it keeps signal variance roughly constant across layers.

Var(w) = 2 / (nin + nout)

He (Kaiming) Initialization

Sets weights with variance 2/nin. Designed for ReLU and its variants, accounting for the fact that ReLU zeros out half the activations, effectively halving the variance.

Var(w) = 2 / nin

Epochs and Batches

Key Definitions

Epoch: One complete pass through the entire training dataset.

Batch size: The number of samples processed before updating weights. A batch size of 1 is stochastic (noisy but fast), while using the full dataset is batch gradient descent (stable but memory-intensive).

Iteration: One weight update step. An epoch contains (dataset size / batch size) iterations.

Learning Rate

📈

The learning rate η controls the step size of each parameter update. A rate that is too large causes the optimizer to overshoot minima and diverge; a rate that is too small makes training painfully slow and can trap the model in poor local minima. Schedulers such as cosine annealing or warm-up strategies gradually adjust η during training to balance speed and stability.

Gradient Clipping

Exploding Gradients

When gradients grow extremely large (common in recurrent networks), the update step can destabilize the model. Gradient clipping caps the gradient norm to a threshold c, ensuring updates stay bounded. This is critical when training on long sequences such as transcribing an entire weather report—without clipping, the model could produce NaN values.

If ‖g‖ > c, then g ← g · (c / ‖g‖)

Optimizers

Vanilla gradient descent computes the gradient over the entire dataset before each update. While each step points in the true direction of steepest descent, a single step is extremely expensive for large datasets. It is rarely used in practice for deep learning because modern datasets can contain billions of examples.

w ← w − η · ∇L(w)

Momentum accumulates a velocity vector that smooths out oscillations and accelerates movement along consistent gradient directions. Imagine a ball rolling downhill: it gains speed in directions of persistent slope and dampens oscillations across narrow valleys. The momentum coefficient β (typically 0.9) determines how much past gradients influence the current step.

vt = β vt−1 + η ∇L(w)
w ← w − vt

RMSProp adapts the learning rate for each parameter by dividing by a running average of recent squared gradients. Parameters with large recent gradients get smaller effective steps, while those with small gradients get larger steps. This helps navigate ravines in the loss landscape where the gradient is steep in one dimension but flat in another.

st = β st−1 + (1 − β)(∇L)2
w ← w − η · ∇L / (√st + ε)

Adam (Adaptive Moment Estimation) combines the benefits of Momentum and RMSProp. It tracks both the first moment (mean) and second moment (uncentered variance) of the gradients, using bias-corrected estimates to update parameters. Adam is the default choice for most Transformer training pipelines because it converges reliably with minimal hyperparameter tuning.

mt = β1 mt−1 + (1 − β1) ∇L
vt = β2 vt−1 + (1 − β2)(∇L)2
m̂t = mt / (1 − β1t)
v̂t = vt / (1 − β2t)
w ← w − η · m̂t / (√v̂t + ε)

Loss Functions

Cross-entropy measures the divergence between a predicted probability distribution and the true distribution. For a classification task with C classes, it penalizes confident wrong predictions heavily. For example, if a model assigns 99% probability to "sunny" when the correct label is "rainy," cross-entropy produces a very large loss.

L = −∑c=1C yc log(&hat;y}c)

Kullback-Leibler divergence quantifies how one probability distribution P diverges from a reference distribution Q. Unlike cross-entropy, KL divergence is asymmetric: DKL(P‖Q) ≠ DKL(Q‖P). It is commonly used in variational inference and when training models to approximate a target distribution.

DKL(P ‖ Q) = ∑ P(x) log(P(x) / Q(x))

These are regression losses used when predicting continuous values, like forecasting a temperature or estimating a recipe's cooking time.

  • MAE (Mean Absolute Error): Average of |y − &hat;y}|. Robust to outliers.
  • MSE (Mean Squared Error): Average of (y − &hat;y})2. Penalizes large errors more heavily.
  • RMSE (Root Mean Squared Error): √MSE. Same units as the target, making it easier to interpret.

Regularization

Dropout

During training, dropout randomly sets a fraction p of neuron activations to zero in each forward pass. This prevents co-adaptation, forcing the network to learn redundant representations. At inference time, all neurons are active but outputs are scaled by (1 − p) to match expected magnitudes. Dropout acts like training an ensemble of sub-networks within a single model.

L1 / L2 / Elastic Net

L1 (Lasso) adds |w| to the loss, encouraging sparsity—many weights become exactly zero, which can serve as automatic feature selection. L2 (Ridge) adds w2, shrinking weights toward zero without eliminating them, which generally improves generalization. Elastic Net combines both with a mixing parameter α, gaining the sparsity of L1 and the stability of L2.

Lreg = L + λ1‖w‖1 + λ2‖w‖22

Evaluation

Home › Part 1: Foundations › Evaluation

Data Splits

Before evaluating any model, you must partition your data to ensure that performance estimates are honest and not inflated by memorization.

Split Purpose Typical Ratio Key Rule
Training Set Used to fit model parameters 70–80% Model sees these examples directly
Validation Set Tune hyperparameters, select checkpoints 10–15% Never used for gradient updates
Test Set Final, unbiased performance estimate 10–15% Used exactly once before deployment
Data Leakage

If any information from the validation or test set influences training (e.g., using test labels to select features), the reported performance will be optimistically biased. Always keep splits strictly separated.

Evaluation Metrics

Metric Definition When to Use
Accuracy Fraction of correct predictions out of all predictions Balanced classes; equal cost of errors
Precision TP / (TP + FP) — of all positive predictions, how many were correct High cost of false positives (e.g., spam filter that might hide important emails)
Recall TP / (TP + FN) — of all actual positives, how many were found High cost of false negatives (e.g., medical diagnosis that must not miss disease)
F1 Score 2 · (Precision · Recall) / (Precision + Recall) Imbalanced classes; need a single number balancing precision and recall
AUC-ROC Area under the curve plotting TPR vs FPR at all thresholds Ranking quality; threshold-agnostic evaluation

Bias-Variance Tradeoff

⚖

Total prediction error decomposes into three sources: bias (error from overly simplistic assumptions), variance (error from sensitivity to training data fluctuations), and irreducible noise. A model that is too simple (high bias) underfits—for instance, fitting a straight line to data that is clearly curved. A model that is too complex (high variance) overfits—memorizing noise such as learning that a particular recipe burned on a Tuesday because of the cook's mood, not the oven temperature. The sweet spot minimizes the sum of bias and variance.

Total Error = Bias2 + Variance + Irreducible Noise

Reducing Bias

  • Use a more expressive model
  • Add features
  • Reduce regularization strength

Reducing Variance

  • Collect more training data
  • Increase regularization
  • Use ensemble methods

Tokenization

Home › Part 2: Vector Representations › Tokenization
✎

What Are Tokens?

Before a model can process text, the raw string must be broken into discrete units called tokens. A token can be a whole word, a subword fragment, or even a single character. The choice of tokenization strategy affects vocabulary size, handling of unseen words, and the granularity of meaning each token carries. For example, the sentence "The robot assistant sorted packages" could be tokenized as whole words [“The”, “robot”, “assistant”, “sorted”, “packages”] or split into subword pieces [“The”, “robot”, “assist”, “##ant”, “sort”, “##ed”, “pack”, “##ages”].

Subword Tokenization Algorithms

BPE starts with a character-level vocabulary and iteratively merges the most frequent adjacent pair of tokens into a new single token. After each merge step, the frequency counts are updated, and the process repeats until the desired vocabulary size is reached. This approach naturally balances vocabulary size with sequence length: common words stay whole, while rare words are broken into shared subword pieces. GPT-2 and many modern LLMs use BPE as their tokenizer.

Step 1: Initialize vocabulary with individual characters (or bytes).
Step 2: Count frequency of every adjacent token pair in the corpus.
Step 3: Merge the most frequent pair into a new token and add it to the vocabulary.
Step 4: Repeat steps 2–3 until the vocabulary reaches the target size.

WordPiece also builds a subword vocabulary through iterative merging, but instead of choosing the most frequent pair, it selects the pair that maximizes the likelihood of the training data under the current language model. In practice, this means evaluating each candidate merge by how much it improves the overall data probability. WordPiece is used by BERT and related models. A key visual cue: WordPiece prefixes continuation tokens with "##" (e.g., "un" + "##doing").

Unigram takes a top-down approach: start with a very large vocabulary (potentially every possible substring) and iteratively remove tokens whose removal causes the smallest decrease in likelihood. At each pruning step, the algorithm computes how much the overall corpus probability would drop if each candidate token were removed, then discards the least important ones. This results in a vocabulary where each token is assigned a probability, and any input can be tokenized in multiple ways—the Viterbi algorithm selects the highest-probability segmentation. SentencePiece uses Unigram as its default algorithm.

Token Embeddings

Home › Part 2: Vector Representations › Token Embeddings

One-Hot Encoding

■

The simplest way to represent a token is a one-hot vector: a vector of length equal to the vocabulary size, with a 1 at the index corresponding to the token and 0 everywhere else. While straightforward, one-hot vectors are extremely sparse and convey no notion of similarity—the vectors for "rain" and "storm" are just as distant as "rain" and "recipe," even though rain and storm are semantically related.

Word2Vec

CBOW (Continuous Bag of Words)

Predicts the center word from surrounding context words. Given the sentence "The ___ forecast predicted rain," CBOW learns to predict "weather" from its neighbors. Training is efficient because averaging the context word vectors smooths gradients.

Skip-Gram

Does the reverse: given a center word, predict the surrounding context words. Starting from "weather," the model tries to predict nearby words like "forecast," "predicted," and "rain." Skip-gram works better for rare words but is slower to train than CBOW.

Word Analogies

↔

A remarkable property of well-trained Word2Vec embeddings is that semantic relationships can be expressed as vector arithmetic. The most famous demonstration finds that the vector from "king" to "queen" parallels the vector from "man" to "woman," because both capture the concept of gender.

vec("king") − vec("man") + vec("woman") ≈ vec("queen")

This works because the embedding space organizes words so that consistent directional offsets encode specific relationships (gender, tense, geography, etc.).

Document Embeddings

Home › Part 2: Vector Representations › Document Embeddings

TF-IDF

📊

Term Frequency–Inverse Document Frequency weighs terms by how important they are to a specific document within a collection. A term that appears frequently in one document but rarely across the collection receives a high weight (e.g., "hurricane" in a weather report, but not in a cookbook). Raw term frequency alone would over-emphasize common words like "the" or "and"; the inverse document frequency component downweights them.

TF-IDF(t, d) = tf(t, d) × log(N / df(t))
where tf(t, d) = count of term t in document d, N = total documents, df(t) = documents containing t

Recurrent Neural Networks (RNN)

🔁

An RNN processes sequences one step at a time, maintaining a hidden state that carries information from previous time steps. At each step, the current input and previous hidden state are combined to produce a new hidden state. This makes RNNs naturally suited for variable-length inputs like sentences. However, vanilla RNNs struggle with long-range dependencies because gradients either vanish or explode when backpropagated through many time steps.

ht = tanh(Whh ht−1 + Wxh xt + bh)

GRU vs LSTM

GRU (Gated Recurrent Unit)

Uses two gates—reset and update—to control information flow. The reset gate decides how much of the past state to forget, while the update gate determines how much new information to incorporate. GRUs are simpler and faster to train than LSTMs because they combine the cell state and hidden state into a single vector.

zt = σ(Wz[ht−1, xt])
rt = σ(Wr[ht−1, xt])
h̃t = tanh(W[ĥt, xt]) where ĥt = rt ⊙ ht−1
ht = (1 − zt) ⊙ ht−1 + zt ⊙ h̃t

LSTM (Long Short-Term Memory)

Uses three gates—forget, input, and output—plus a separate cell state that acts as a memory highway. The forget gate decides what to discard from the cell state; the input gate decides what new information to store; the output gate decides what part of the cell state to expose as the hidden state. LSTMs generally outperform GRUs on tasks requiring very long-range dependencies.

ft = σ(Wf[ht−1, xt])
it = σ(Wi[ht−1, xt])
C̃t = tanh(WC[ht−1, xt])
Ct = ft ⊙ Ct−1 + it ⊙ C̃t
ot = σ(Wo[ht−1, xt])
ht = ot ⊙ tanh(Ct)

ELMo

🎨

Embeddings from Language Models (ELMo) was a breakthrough in contextual word representations. Unlike Word2Vec, which assigns a single static vector to each word, ELMo generates different vectors for the same word depending on its context. The word "bank" in "river bank" and "bank account" receives distinct embeddings because ELMo reads the entire sentence bidirectionally using stacked bidirectional LSTMs. The final representation is a weighted combination of all LSTM layer outputs, where the weights are learned for each downstream task.

Similarity Measures

Measure Formula Range Use Case
Cosine Similarity cos(a, b) = (a · b) / (‖a‖ · ‖b‖) [-1, 1] Directional similarity regardless of magnitude; standard for text embeddings
Euclidean Distance d(a, b) = √∑(ai − bi)2 [0, ∞) Absolute distance in vector space; sensitive to magnitude
Dot Product a · b = ∑ ai bi (−∞, ∞) Fast to compute; used when vectors are pre-normalized
Manhattan Distance d(a, b) = ∑|ai − bi| [0, ∞) Robust to outliers in high-dimensional spaces

t-SNE

🕷

t-Distributed Stochastic Neighbor Embedding (t-SNE) is a dimensionality reduction technique designed for visualization. It maps high-dimensional embeddings (e.g., 768-dimensional BERT vectors) down to 2D or 3D while preserving local neighborhoods—points that are close in the original space tend to remain close in the plot. t-SNE uses a Student's t-distribution in the low-dimensional space to avoid the "crowding problem" that plagues earlier methods. However, it is non-parametric (cannot be applied to new points without re-running), and distances between distant clusters are not meaningful.

Locality-Sensitive Hashing (LSH)

🔍

LSH is a technique for approximate nearest-neighbor search in massive datasets. Instead of comparing a query vector against every stored vector (which is O(n)), LSH hashes vectors into buckets such that similar vectors are likely to land in the same bucket. You only compare the query against vectors in the same bucket, dramatically reducing search time. This is essential for retrieval systems that must find relevant documents among billions in milliseconds—for instance, a search engine matching a user's query to cached weather reports.

LSH in Practice

Common LSH families include random hyperplane hashing for cosine similarity and min-hash for Jaccard similarity. Multiple hash tables with different random projections are combined to reduce false negatives at the cost of extra storage.

Self-Attention

Home › Part 3: The Transformer › Self-Attention
💡

Query, Key, Value

Self-attention borrows the retrieve-and-read pattern from information retrieval. Each token in a sequence produces three vectors by multiplying its embedding with learned projection matrices:

  • Query (Q) — What information am I looking for? Like typing "robot assistant features" into a search bar.
  • Key (K) — What information do I contain? Like the title and tags on a help article.
  • Value (V) — The actual content I contribute if selected. Like the full text of the article.

The attention score between two tokens is computed as the dot product of the query of one token with the key of another. High dot products indicate relevance, so tokens whose queries align with certain keys will attend strongly to those keys' values.

Scaled Dot-Product Attention

⚙

The raw dot products between Q and K grow with the dimension dk. When dk is large, softmax pushes its inputs toward extreme values (very close to 0 or 1), yielding tiny gradients. Dividing by √dk keeps the variance of the dot products approximately 1 regardless of dimension, ensuring stable gradient flow.

Attention(Q, K, V) = softmax(QKT / √dk) V

Why do we divide the attention scores by √dk?

Multi-Head Attention

🔸

Instead of performing a single attention function, multi-head attention runs several attention operations in parallel, each with its own learned Q, K, V projections. Each "head" can focus on different types of relationships—one head might attend to syntactic dependencies (subject-verb agreement), another to coreference (linking "it" back to "robot"), and another to positional proximity. The outputs of all heads are concatenated and linearly projected to produce the final result.

MultiHead(Q, K, V) = Concat(head1, ..., headh) WO
where headi = Attention(Q WiQ, K WiK, V WiV)

Architecture

Home › Part 3: The Transformer › Architecture

End-to-End Flow

1. Tokenize Input — The raw text (e.g., "What will the weather be tomorrow?") is split into tokens using the chosen tokenizer.
2. Embed Tokens — Each token ID is mapped to a dense embedding vector, and a positional encoding is added so the model knows the order of tokens.
3. Encoder Processes Input — The embedded sequence passes through N stacked encoder layers. Each layer applies multi-head self-attention followed by a feed-forward network, with residual connections and layer normalization around each sub-layer.
4. Decoder Generates Output — The decoder attends to its own previously generated tokens (via masked self-attention) and to the encoder's output (via cross-attention), then passes through its own feed-forward network to predict the next token.
5. Output Projection — The decoder's final hidden states are projected through a linear layer followed by softmax over the vocabulary to produce a probability distribution for the next token.
6. Decode Token — The highest-probability token (or a sampled token) is appended to the output sequence, and the process repeats autoregressively until an end-of-sequence token is produced.
Part 3

Encoder & Decoder

Home › Part 3: The Transformer › Encoder & Decoder

Encoder Components

▶

Each encoder layer contains two sub-layers: (1) a multi-head self-attention mechanism and (2) a position-wise feed-forward network. A residual connection wraps each sub-layer, and layer normalization is applied to the sum of the input and sub-layer output: LayerNorm(x + Sublayer(x)). This "Add & Norm" pattern stabilizes training in deep stacks. The feed-forward network is typically two linear transformations with a GELU or ReLU activation in between, applied identically to each position.

FFN(x) = max(0, xW1 + b1)W2 + b2
(or with GELU: FFN(x) = GELU(xW1 + b1)W2 + b2)

Decoder Components

◀

Masked Self-Attention

In the decoder, self-attention is masked so that position i can only attend to positions ≤ i. This prevents the model from peeking at future tokens during training. Without masking, the decoder could trivially copy the next token rather than learning to predict it. The mask is implemented by setting future positions to −∞ before applying softmax, which drives their attention weights to zero.

🔃

Cross-Attention

The second attention sub-layer in each decoder layer is cross-attention. Here, the queries come from the decoder's previous layer, while keys and values come from the encoder's output. This allows the decoder to focus on relevant parts of the input sequence while generating each output token. For instance, when translating "robot assistant" to another language, cross-attention lets the decoder look up the encoder's representation of "robot" when deciding how to begin the translation.

🛠

Feed-Forward Network

Each decoder layer also contains a position-wise FFN identical in structure to the encoder's. After cross-attention, the FFN adds nonlinearity and additional capacity, enabling the model to refine its intermediate representations before passing them to the next layer.

Layer Normalization

📈

Layer normalization stabilizes training by normalizing the features across the embedding dimension for each token independently. Unlike batch normalization, it does not depend on batch statistics, making it well-suited for variable-length sequences and small batch sizes. There are two common placements: Post-LN (original Transformer) applies LayerNorm after the residual connection, while Pre-LN applies it before the sub-layer. Pre-LN is generally more stable for very deep models.

μ = (1/d) ∑i=1d xi
σ2 = (1/d) ∑i=1d (xi − μ)2
LayerNorm(x) = γ ⊙ (x − μ) / √(σ2 + ε) + β

Position Awareness

Home › Part 3: The Transformer › Position Awareness
Why Position Matters

Self-attention is permutation-invariant: it treats the input as a bag of vectors with no inherent order. The sentences "The robot helped the human" and "The human helped the robot" would produce identical attention patterns without positional signals. Position encodings inject order information so the model can distinguish between the two.

The simplest approach: treat position index as a categorical feature and learn an embedding table of size (max_seq_len, dmodel). Each position 0, 1, 2, ... gets its own learned vector that is added to the token embedding. Used in BERT and GPT. Drawback: cannot extrapolate to sequences longer than those seen during training.

The original Transformer uses fixed sinusoidal functions of different frequencies across dimensions. Each dimension of the positional encoding corresponds to a sinusoid whose wavelength ranges from 2π to 2π · 10000. This design allows the model to learn relative positions because for any fixed offset k, PE(pos + k) can be expressed as a linear function of PE(pos).

PE(pos, 2i) = sin(pos / 100002i/dmodel)
PE(pos, 2i+1) = cos(pos / 100002i/dmodel)

Rotary Position Embeddings encode position by rotating pairs of dimensions in the query and key vectors. The rotation angle is a function of the token's position, so the dot product between a query at position m and a key at position n naturally encodes the relative distance (m − n). RoPE is used in LLaMA, PaLM, and many modern LLMs because it provides clean relative position information without adding anything to the value vectors.

ALiBi takes a different approach: instead of adding positional encodings to embeddings, it adds a linearly decreasing penalty to attention scores based on distance. Tokens that are far apart receive a larger negative bias, reducing their attention weight. ALiBi enables length extrapolation—models trained on short sequences can generalize to much longer ones without retraining. This is useful when processing lengthy documents like technical manuals.

Architecture Variants

Home › Part 3: The Transformer › Architecture Variants

Encoder-Only (BERT)

Uses only the encoder stack with bidirectional attention—every token can attend to every other token. Ideal for tasks that require understanding the full input, such as classification, named entity recognition, and question answering (extractive). BERT is pretrained with masked language modeling, where random tokens are masked and the model predicts them from context.

Decoder-Only (GPT)

Uses only the decoder stack with causal (masked) attention—each token can only attend to itself and earlier tokens. Ideal for text generation. GPT is pretrained with next-token prediction and can be prompted to perform many tasks without finetuning. Most modern LLMs (GPT-4, LLaMA, Mistral) use the decoder-only architecture.

Encoder-Decoder (T5)

Retains both the encoder and decoder. The encoder processes the input bidirectionally, and the decoder generates output autoregressively with cross-attention to the encoder. Well-suited for seq2seq tasks like translation, summarization, and structured generation. T5 frames all tasks as text-to-text: both input and output are strings.

Mixture of Experts (MoE)

🌐

MoE scales model capacity without proportionally increasing computation per token. The feed-forward network in each Transformer layer is replaced with multiple parallel "expert" FFNs and a learned router that selects a small subset (often 1 or 2) of experts per token. Each token only activates a fraction of the total parameters, keeping FLOPs manageable while total parameter count can be enormous. For example, a model with 8 experts and top-2 routing uses only 2 experts per token but has 8× the FFN capacity. Challenges include load balancing (ensuring all experts receive roughly equal traffic) and training stability.

Attention Speedup

Home › Part 3: The Transformer › Attention Speedup
⚡

Standard self-attention has O(n2) time and memory complexity with respect to sequence length n, because it computes and stores an n×n attention matrix. For long sequences, this becomes a bottleneck. Several methods reduce this cost.

Sparse Attention

🗃

Sparse attention restricts each token to attend to only a subset of positions rather than all n tokens. Common patterns include local windows (attend only to nearby tokens), strided patterns (attend to every k-th token), and combinations that allow a mix of local and global attention. Longformer uses sliding window attention with a few global tokens; BigBird adds random attention on top. These reduce complexity to O(n · w) where w is the window size.

Low-Rank Attention

🔹

Low-rank methods (e.g., Linformer) project the n×d key and value matrices down to k×d using learned projection matrices, where k << n. The attention matrix is then k×n instead of n×n, reducing complexity to O(nk). This works well in practice because attention matrices tend to be low-rank—most of their information is captured by a small number of principal components.

Flash Attention

🔥

Flash Attention is an exact (not approximate) attention algorithm that reduces memory reads/writes by tiling the computation to fit within GPU SRAM. Instead of materializing the full n×n attention matrix in high-bandwidth memory (HBM), it computes the softmax row-by-row in smaller blocks, writing only the final output back to HBM. This yields the same mathematical result as standard attention but runs 2–4× faster and uses O(n) memory instead of O(n2). Flash Attention 2 further optimizes parallelism and work partitioning across GPU warps.

When to Use Which

Flash Attention: Drop-in replacement when you have compatible hardware (Ampere+ GPUs). Sparse Attention: When sequences exceed even Flash Attention's capacity. Low-Rank: When approximate attention is acceptable and you need consistent O(nk) scaling.

Part 3

Interpretability

Home › Part 3: The Transformer › Interpretability

Attention Visualization

👁

One of the most intuitive ways to inspect a Transformer is to visualize the attention weight matrices. By displaying which tokens attend most strongly to which others, you can gain insight into what the model considers relevant. For example, in a sentence about a robot assistant, you might observe that the pronoun "it" strongly attends to "robot," suggesting the model has learned coreference. However, attention weights are not guarantees of model reasoning—they reflect correlation, not necessarily causation.

Probing

🔬

Probing involves training small auxiliary classifiers (probes) on the intermediate representations of a frozen model to test whether specific linguistic properties are encoded. For instance, a probe might predict the part-of-speech tag of a token from its layer-6 representation. If the probe achieves high accuracy, the representation encodes that property. Probing has revealed that lower Transformer layers tend to encode surface-level features (POS tags), middle layers encode syntactic structure, and upper layers encode semantic information.

Model-Agnostic Methods

🔎

These techniques do not require access to internal model states and can be applied to any model. SHAP (SHapley Additive exPlanations) assigns each input feature an importance value based on game-theoretic Shapley values. LIME (Local Interpretable Model-agnostic Explanations) fits a simple interpretable model locally around a specific prediction to approximate which features drove that prediction. Both methods help answer "Why did the model produce this output?" without needing to understand the model's internals.

Part 4

Fundamentals

Home › Part 4: Large Language Models › Fundamentals

Response Generation

💬

At its core, an LLM generates responses one token at a time. Given a prompt, the model computes a probability distribution over the entire vocabulary for the next token, selects a token according to some sampling strategy, appends it to the context, and repeats. This autoregressive loop continues until the model produces a special end-of-sequence token or a maximum length is reached. The quality and diversity of the output depend heavily on how tokens are sampled from the distribution.

Temperature Sampling

🌡

Temperature (τ) controls the sharpness of the probability distribution before sampling. The logits are divided by τ before applying softmax:

P(xi) = exp(logiti / τ) / ∑j exp(logitj / τ)

Low Temperature (τ near 0)

Distribution becomes peaked; the highest-probability token dominates. Outputs are deterministic and focused but may be repetitive.

High Temperature (τ > 1)

Distribution flattens; all tokens become more equally likely. Outputs are diverse and creative but may be incoherent.

Top-k and Top-p Sampling

Top-k Sampling

Only the k most probable tokens are considered; the rest are set to zero probability before renormalization and sampling. A small k (e.g., 10) keeps output focused, while a large k (e.g., 100) allows more variety. The limitation is that k is fixed regardless of how peaked or flat the distribution is.

Top-p (Nucleus) Sampling

Instead of a fixed count, top-p selects the smallest set of tokens whose cumulative probability exceeds p (e.g., 0.9). This adapts to the distribution shape: when the model is confident, only a few tokens are needed; when uncertain, more tokens are included. Top-p is generally preferred over top-k because of this adaptivity.

Context Window

📖

The context window is the maximum number of tokens the model can process in a single forward pass. It is determined by the positional encoding scheme and the memory available for the attention matrices. Everything outside the context window is invisible to the model. For example, a model with a 4K token context can read roughly 3,000 words of input and output combined. Modern models extend this to 32K through 1M tokens, but longer contexts increase compute cost quadratically without Flash Attention.

Part 4

Pretraining

Home › Part 4: Large Language Models › Pretraining

Data Mixtures

📚

LLMs are pretrained on massive, diverse text corpora. The mixture typically includes:

  • Internet text (Common Crawl, web scrapes) — enormous volume but noisy and repetitive. Often filtered for quality and deduplicated.
  • Books — long-form, coherent prose that teaches sustained argumentation, narrative structure, and domain knowledge.
  • Code (GitHub repositories) — teaches logical reasoning, structured thinking, and precise syntax. Code exposure strongly correlates with improved reasoning benchmarks.
  • Curated sources (Wikipedia, scientific papers) — high-quality factual content that improves accuracy on knowledge-intensive tasks.

The exact ratios are closely guarded by most organizations, but research suggests that even small amounts of high-quality data can disproportionately improve performance.

Next Token Prediction

🎯

The core pretraining objective for autoregressive LLMs is next token prediction: given tokens 1 through t-1, predict token t. The loss is the cross-entropy between the model's predicted distribution and the actual next token. By repeating this over trillions of tokens, the model learns grammar, facts, reasoning patterns, and stylistic conventions. Despite its simplicity, next token prediction produces remarkably capable models because the diversity of the training data forces the model to develop rich internal representations.

Autoregressive / Causal Language Modeling

🕐

In causal language modeling, each token is predicted conditioned only on preceding tokens (left-to-right). The attention mask ensures no future information leaks. This is the standard pretraining paradigm for GPT-family models. The key advantage is that the same architecture and objective used during pretraining also works at inference time; there is no mismatch between training and deployment. The disadvantage is that the model never sees right-side context, which limits its ability to perform bidirectional understanding tasks without careful prompting.

Part 4

Prompt Engineering

Home › Part 4: Large Language Models › Prompt Engineering

Prompt Structure

📝

A well-designed prompt typically contains several components:

  • Context — Background information the model needs. "You are a robot assistant helping a user set up their smart home."
  • Instructions — Specific task description. "Classify each device as sensor or actuator."
  • Input — The actual data to process. "thermometer, motor, camera, speaker"
  • Examples (few-shot) — Input-output pairs demonstrating the desired format.
  • Constraints — Rules the model must follow. "Respond only in JSON. Do not include explanations."

In-Context Learning

Zero-Shot

The model performs the task with only instructions and no examples. Relies entirely on knowledge acquired during pretraining. Works well for simple, common tasks like "Translate to French: good morning."

Few-Shot

A handful of input-output examples are included in the prompt before the actual query. The model learns the pattern from these examples without any weight updates. Typically 3 to 10 examples suffice; more examples improve accuracy but consume context window space.

Chain of Thought (CoT)

🔗

Chain of Thought prompting asks the model to reason step-by-step before producing a final answer. Instead of prompting "What is 23 times 47?", you prompt "What is 23 times 47? Let us think step by step." The model then generates intermediate reasoning steps. CoT dramatically improves performance on arithmetic, logic, and multi-step reasoning tasks because the model's computation is distributed across multiple tokens rather than forced into a single prediction.

Self-Consistency

🔁

Self-consistency extends CoT by sampling multiple independent reasoning paths for the same prompt (using temperature greater than 0) and selecting the most common final answer by majority vote. If most reasoning paths arrive at the same conclusion, it is more likely to be correct. This trades compute for reliability and is especially effective on tasks with a discrete set of answers.

Tree of Thoughts

🌳

Tree of Thoughts generalizes CoT by exploring multiple reasoning paths simultaneously in a tree structure. At each step, the model generates several candidate "thoughts" (partial solutions), evaluates them, and keeps the most promising branches. This is analogous to how a chess engine explores multiple moves ahead and prunes weak lines. Tree of Thoughts excels at planning tasks, such as composing a multi-course recipe where ingredient choices in one dish constrain options for others.

Prompt Risks

LLMs can generate confident-sounding statements that are factually incorrect. This happens because the model optimizes for plausible continuations, not truth. For example, a model might confidently describe a nonexistent weather phenomenon because the words sound scientifically plausible. Mitigation strategies include grounding outputs in retrieved documents (RAG), asking the model to cite sources, and cross-checking critical claims.

Training data reflects societal biases (gender, racial, cultural). LLMs can reproduce and amplify these biases in their outputs. For instance, a model might associate "nurse" with female pronouns and "surgeon" with male pronouns simply because this pattern appears frequently in its training corpus. Careful dataset curation, debiasing techniques, and output filtering can reduce but not eliminate bias.

Jailbreaking refers to crafting prompts that bypass safety filters and cause the model to produce harmful, unethical, or restricted content. Common techniques include role-playing scenarios, encoding harmful requests in puzzles or translations, and multi-turn manipulation that gradually pushes the model toward violating its constraints. Defense is an ongoing cat-and-mouse game; no system is perfectly robust.

Part 4

Finetuning

Home › Part 4: Large Language Models › Finetuning

Supervised Finetuning (SFT)

🎓

After pretraining, the model can continue text but does not naturally follow instructions. SFT trains the model on curated (prompt, response) pairs so it learns to behave as a helpful assistant. For example, given the prompt "Explain how a robot assistant processes voice commands," the target response would be a clear, informative answer. The model is trained with standard cross-entropy loss on these pairs, learning the style and format of desirable interactions.

Parameter-Efficient Finetuning

LoRA freezes the pretrained weight matrix W and learns a low-rank decomposition of the update: W' = W + BA, where B is d x r and A is r x d with r much smaller than d. Only A and B are trained, dramatically reducing the number of trainable parameters (often by 99%+). At inference, the low-rank update can be merged back into W for zero additional latency.

W' = W + BA
where B ∈ Rd×r, A ∈ Rr×d, r << d
Trainable params: 2 × d × r (vs d × d for full finetuning)

QLoRA combines quantization with LoRA. The pretrained model is stored in 4-bit NormalFloat format (a quantization scheme optimized for normally-distributed weights), and the LoRA adapters are trained in full precision (bf16). This enables finetuning a 65B parameter model on a single 48GB GPU. Key innovations include double quantization (quantizing the quantization constants themselves) and paged optimizers to handle memory spikes.

Prefix tuning prepends a small number of learnable "virtual tokens" (the prefix) to each Transformer layer's key and value matrices. These tokens are not part of the vocabulary; they are free parameters optimized during finetuning. The pretrained weights remain frozen. Because the prefix length is short (typically 10-20 tokens), only a tiny fraction of parameters are trained. However, the prefix occupies part of the context window at inference time.

Adapters are small bottleneck modules inserted between the Transformer layers. Each adapter consists of a down-projection (d to r), a nonlinearity, and an up-projection (r to d), with a residual connection around it. Only the adapter parameters are trained; the base model is frozen. The residual connection ensures that if the adapter learns nothing useful, the original layer output passes through unchanged, making training stable.

Adapter(x) = x + f(Wup · σ(Wdown · x))
Part 4

Preference Alignment

Home › Part 4: Large Language Models › Preference Alignment

Core Concepts

🏆

SFT teaches the model what good responses look like, but it does not ensure the model avoids harmful or low-quality outputs. Preference alignment goes further by training the model to prefer some responses over others based on human judgment. Two key components are:

  • Reward Model — A separate model trained to assign scalar scores to responses, reflecting how desirable they are according to human raters.
  • Human Feedback — Annotators rank multiple model outputs for the same prompt. These rankings provide the training signal for the reward model.

RLHF (Reinforcement Learning from Human Feedback)

Step 1: Reward Model Training — Given a prompt and several ranked responses, the reward model learns to assign higher scores to higher-ranked outputs. It is trained with a Bradley-Terry pairwise ranking loss: the difference in reward between a preferred and dispreferred response should be large.
Step 2: Policy Optimization (PPO) — The language model (policy) generates responses, which are scored by the frozen reward model. Proximal Policy Optimization (PPO) updates the policy to maximize reward while staying close to the original SFT model (via a KL divergence penalty) to prevent reward hacking, where the model exploits reward model blind spots.
Objective = E[r(x, y)] - β · KL(πθ ‖ πref)
where r is the reward model, πθ is the current policy, πref is the reference (SFT) model, and β controls the KL penalty strength

DPO (Direct Preference Optimization)

⚡

DPO eliminates the separate reward model entirely. Instead of training a reward model and then using RL to optimize it, DPO directly optimizes the language model on preference pairs using a closed-form solution derived from the KL-constrained reward maximization objective. Given a preferred response yw and dispreferred response yl for the same prompt x, DPO increases the likelihood of yw relative to yl.

LDPO = -E[log σ(β log(πθ(yw|x) / πref(yw|x)) - β log(πθ(yl|x) / πref(yl|x)))]
where σ is the sigmoid function and β controls the penalty for deviation from the reference policy

Model Compression

Home › Part 4: Large Language Models › Model Compression

Distillation

🎟

Knowledge distillation transfers knowledge from a large "teacher" model to a smaller "student" model. The student is trained to mimic the teacher's output probability distribution (soft labels) rather than just the hard ground-truth labels. The teacher's distribution contains "dark knowledge" — information about which incorrect answers the teacher considers plausible — which gives the student richer training signal than hard labels alone. Temperature is raised in the teacher's softmax to soften the distribution and expose these inter-class relationships.

Ldistill = α · KL(softmax(zt / τ) ‖ softmax(zs / τ)) + (1 - α) · LCE(y, zs)
where zt = teacher logits, zs = student logits, τ = temperature, α = mixing weight

Quantization

Post-Training Quantization (PTQ)

Quantize a pretrained model without retraining. Fast and simple, but can degrade accuracy, especially at very low bit-widths. Common approaches include weight-only quantization and weight-plus-activation quantization.

Quantization-Aware Training (QAT)

Simulates quantization effects during training by inserting fake quantization nodes into the forward pass. The model learns to be robust to the precision loss. More accurate than PTQ but requires training infrastructure and data.

INT8 and INT4 Quantization

📊

INT8 represents each weight with 8-bit integers (256 values), reducing memory by 4x compared to FP32 with typically minor accuracy loss. INT4 pushes further to 4-bit integers (16 values), reducing memory by 8x but requiring careful calibration. INT4 is commonly applied only to weights (weight-only quantization) while keeping activations in higher precision to maintain output quality.

Blockwise Quantization

▦

Instead of using a single scale factor for the entire weight matrix, blockwise quantization divides weights into small blocks (e.g., 64 elements) and computes a separate scale factor per block. This provides finer granularity and better preserves the dynamic range of outlier weights, which are common in LLMs and critical for performance.

Mixed Precision Training

💸

Mixed precision training uses FP16 or BF16 for most forward and backward computations while maintaining a master copy of weights in FP32 for stability. Gradients computed in lower precision can underflow; loss scaling multiplies the loss by a large constant before backpropagation and divides gradients by the same constant after, keeping small gradient values in the representable range. Modern GPUs with Tensor Cores accelerate FP16/BF16 operations by 2-8x over FP32.

QLoRA Details

🔧

QLoRA combines 4-bit quantization with LoRA adapters to enable finetuning of large models on consumer hardware. Three key innovations make this work:

  • 4-bit NormalFloat (NF4) — A data type optimized for normally-distributed neural network weights, providing better information density than uniform INT4.
  • Double Quantization — Quantizes the quantization constants themselves, saving approximately 0.37 bits per parameter.
  • Paged Optimizers — Uses CPU memory to offload optimizer states during memory spikes, preventing out-of-memory errors on long sequences.

LLM.int8()

🔎

LLM.int8() is a mixed-precision quantization method for inference. It identifies outlier feature dimensions (values with magnitude greater than a threshold, typically 6.0) and computes those dimensions in FP16 while the remaining 99.9% of dimensions use INT8 matrix multiplication. This is necessary because LLMs develop systematic outliers in specific feature dimensions at scale (beyond 6B parameters), and naively quantizing these outliers destroys model quality.

Evaluation Framework

Home › Part 5: Applications › Evaluation Framework

NLU vs NLG

NLU (Natural Language Understanding)

Tasks where the model must comprehend and categorize existing text. The output is typically a label, span, or structured extraction. Examples include sentiment analysis, intent detection, and named entity recognition. Evaluation is often straightforward because there is a single correct answer.

NLG (Natural Language Generation)

Tasks where the model must produce fluent, coherent text. The output is open-ended, making evaluation harder because many valid responses exist. Examples include translation, summarization, and open-ended dialogue. Evaluation often relies on overlap metrics or model-based judges.

Classification Tasks

Task Description Example
Sentiment Analysis Classify the emotional tone of text as positive, negative, or neutral "This robot assistant exceeded my expectations!" → Positive
Intent Detection Identify the user's goal from an utterance "Set a timer for 10 minutes" → SetTimer intent
Named Entity Recognition Locate and classify entity mentions in text "Forecast for Paris on Tuesday" → Paris:Location, Tuesday:Date
POS Tagging Assign a grammatical part-of-speech to each token "The/DT robot/NN runs/VBZ"

Generation Tasks

Task Description Example
Machine Translation Translate text from one language to another "How is the weather?" → "Comment est la météo?"
Question Answering Produce an answer given a question and optional context Q: "What temperature does water boil?" A: "100 degrees Celsius at sea level"
Summarization Condense a long document into a shorter version preserving key information 10-page weather report → "Heavy rain expected Tuesday through Thursday"

Common Benchmarks

GLUE

The General Language Understanding Evaluation benchmark includes nine diverse NLU tasks spanning sentiment analysis, textual entailment, and paraphrase detection. It was the standard for encoder models like BERT but is now largely saturated; modern models exceed human-level performance on most GLUE tasks.

SuperGLUE

A harder successor to GLUE with more challenging tasks including coreference resolution (WSC), question answering (MultiRC), and word sense disambiguation. Designed to push models beyond GLUE-level performance.

MMLU

Massive Multitask Language Understanding spans 57 subjects ranging from elementary math to professional law and clinical knowledge. It tests both factual recall and reasoning, making it a popular benchmark for large language models.

HELM

Holistic Evaluation of Language Models evaluates models across 42 scenarios using 7 metrics (accuracy, calibration, robustness, fairness, bias, toxicity, efficiency). Unlike single-score benchmarks, HELM provides a multi-dimensional profile of model capabilities and risks.

Part 5

Classification Tasks

Home › Part 5: Applications › Classification Tasks

Sentiment Extraction

😊

While basic sentiment classification labels an entire document, sentiment extraction (or aspect-based sentiment analysis) identifies the sentiment toward specific aspects. For a restaurant review like "The pasta was excellent but the service was slow," sentiment extraction would mark "pasta" as positive and "service" as negative. This finer granularity is essential for actionable feedback analysis, helping businesses understand exactly what to improve.

Named Entity Recognition (NER)

🔍

NER locates and classifies named entities in text into predefined categories such as Person, Organization, Location, Date, and Product. Modern approaches use token-level classification where each token receives a label using the BIO tagging scheme (B- = beginning of entity, I- = inside entity, O = outside any entity). For example, in "Apple released the iPhone in Cupertino on Tuesday," the model should tag "Apple" as B-ORG, "iPhone" as B-PRODUCT, "Cupertino" as B-LOC, and "Tuesday" as B-DATE.

NER in Practice

NER is a foundational component in many pipelines: information extraction, knowledge graph construction, and document search. In a weather forecasting system, NER could extract city names and dates to trigger the appropriate data lookups automatically.

Part 5

Generation Tasks

Home › Part 5: Applications › Generation Tasks

Machine Translation

🌐

Machine translation converts text from a source language to a target language while preserving meaning. Modern neural translation uses encoder-decoder or decoder-only architectures trained on parallel corpora. A key challenge is handling rare words, idiomatic expressions, and cultural context that does not translate literally.

BLEU Metric

📊

BLEU (Bilingual Evaluation Understudy) measures the overlap of n-grams between the model output and one or more reference translations. It computes a modified precision for each n-gram order (typically 1 through 4) and combines them with a brevity penalty that penalizes overly short outputs. BLEU is fast and reproducible but has known limitations: it does not capture synonymy, reordering well, or semantic adequacy.

BLEU = BP · exp(∑n=1N wn log pn)
BP = min(1, exp(1 - r/c))
where pn = modified n-gram precision, r = reference length, c = candidate length

Summarization

📄

Summarization produces a condensed version of a longer document while retaining the most important information. Abstractive summarization generates novel text (not just copying sentences), while extractive summarization selects and combines existing sentences. Modern models typically perform abstractive summarization, which requires deep understanding of the source text.

ROUGE Metric

📈

ROUGE (Recall-Oriented Understudy for Gisting Evaluation) measures recall of n-grams from the reference summary in the generated summary. Common variants include:

  • ROUGE-1 — Unigram overlap
  • ROUGE-2 — Bigram overlap
  • ROUGE-L — Longest common subsequence, which captures word order to some degree

Unlike BLEU (which measures precision), ROUGE measures recall: how much of the reference content appears in the generated output. High ROUGE scores indicate the summary covers the key points but do not guarantee fluency or factual accuracy.

Retrieval-Augmented Generation (RAG)

📚

RAG addresses two major LLM limitations: hallucination and outdated knowledge. Instead of relying solely on parametric memory, RAG first retrieves relevant documents from an external knowledge base and then conditions the model's generation on those documents. This grounds the output in verifiable sources and allows the knowledge base to be updated without retraining the model.

Step 1: Index Documents — The knowledge base is split into chunks, each chunk is embedded into a vector, and all vectors are stored in an index (e.g., a vector database) that supports fast similarity search.
Step 2: Retrieve Relevant Chunks — When a query arrives, it is embedded using the same embedding model. The index returns the top-k chunks whose embeddings are most similar to the query embedding (using cosine similarity or approximate nearest-neighbor search).
Step 3: Augment the Prompt — The retrieved chunks are inserted into the LLM prompt alongside the original query, typically with an instruction like "Answer the question using only the provided context."
Step 4: Generate Response — The LLM reads the augmented prompt and generates an answer that is grounded in the retrieved documents, reducing the chance of hallucination.
RAG Limitations

RAG is only as good as its retrieval step. If the relevant documents are not found, the model cannot use them. Noisy or contradictory retrieved chunks can confuse the model. Chunk size, embedding quality, and the number of retrieved documents (k) are critical hyperparameters that require careful tuning.
