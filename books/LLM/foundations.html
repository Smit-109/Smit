Neural Networks, Training, and Evaluation

1.1 Neural Networks

1.1.1 Overview

Neural Network (NN)

A model inspired by how the brain works. Contains neurons connected to one another, separated by nonlinearities, which allow them to learn complex patterns.

Feed-Forward Neural Network (FFNN)

Composed of three types of layers:

  • Input layer: Receives input as a vector of numbers
  • Hidden layers: Transform input using multiplications, additions, and nonlinearities
  • Output layer: Outputs final vector representing model prediction
Input → [h1] → [h2] → ... → [hk] → Output

Note: A network with small number of hidden layers (typically k ≤ 3) is called a shallow network.

1.1.2 Components

Input Layer

Represented by an embedding vector i ∈ ℝm. Examples by modality:

ModalityDescriptionExample
TextText divided into entities, each represented by a vectorOne-hot encoding
AudioAudio divided into time segmentsMel-spectrograms
ImageEach pixel represented by a vectorRGB encoding

Hidden Layer

A hidden layer allows the network to learn complex patterns. It consists of:

  • Weights w: Learned parameters multiplied with input
  • Biases b: Learned parameters added to result
  • Activation function A: Introduces nonlinearities

Activation Functions

NameFunctionOutput Range
Sigmoidσ(z) = 1/(1+e-z)(0, 1)
Tanhtanh(z) = (ez-e-z)/(ez+e-z)(-1, 1)
ReLUmax(0, z)[0, +∞)

GELUz·Φ(z), Φ ~ N(0,1)(-0.17, +∞)

Gated Linear Unit (GLU)

A(z) = A1(z) ⊙ A2(z)

Variants:

  • GLU: (zW+b) ⊙ σ(zV+c)
  • SwiGLU: (zW+b) ⊙ SiLU(zV+c)
  • ReGLU: ReLU(zW+b) ⊙ (zV+c)

Output Layer

Returns final prediction via vector o ∈ ℝn:

  • Classification: Softmax to convert to probability distribution
  • Regression: Returns o as is
softmax(o)i = eoi / Σj eoj

1.2 Training

1.2.1 Parameter Learning

Weight Initialization

Critical for training convergence. Common methods:

MethodActivationInitialization
Xavier/GlorotSigmoid, TanhUniform [-√(6/(ni+no)), √(6/(ni+no))]
HeReLUNormal with σ = √(2/ni)

Where ni = input neurons, no = output neurons

Epoch and Batch

  • Training size N: Number of observations in training set
  • Batch size b: Observations processed at each pass
  • Steps per epoch: s = N/b
N = b × s

Loss Function

Quantifies how close predictions ŷ are to targets y. Higher values = worse performance.

Labels

  • Hard label: Observation belongs to class (yi = 1) or not (yi = 0)
  • Soft label: Probability yi ∈ [0, 1] of belonging to class

Backpropagation

Minimizes loss by adjusting parameters through gradient calculation:

  1. Forward step: Compute predictions ŷ and loss L(ŷ, y)
  2. Backward step: Compute ∂L/∂θ for each parameter
  3. Update step: θ ← θ - η ∂L/∂θ

Gradient Clipping

Prevents exploding gradients by capping gradients at ceiling C:

∇θclipped = clip(∇θ, -C, C)

Learning Rate (η)

RateEffect
Too LowSlow convergence, stuck in non-optimal region
Just RightOptimal convergence
Too HighMay diverge and miss optimal region

Warmup

Use lower learning rate during first sw steps to prevent early overfitting:

η(s) = C × min(1, s/sw3)

1.2.2 Optimizers

Gradient Descent

θt+1 = θt - η ∇L(θt)

Momentum

Accelerates convergence by dampening oscillations:

vt+1 = βvt + (1-β)∇L(θt)
θt+1 = θt - ηvt+1

β typically set to 0.9

RMSProp

Adaptive learning rates using moving average of squared gradients:

vt+1 = βvt + (1-β)(∇L(θt))2
θt+1 = θt - η ∇L(θt) / √(vt+1+ε)

Adam

Combines benefits of Momentum and RMSProp:

mt+1 = β1mt + (1-β1)∇L(θt)
vt+1 = β2vt + (1-β2)(∇L(θt))2
θt+1 = θt - η mt+1/(√vt+1+ε)

Typical values: β1=0.9, β2=0.999, η=0.001

AdamW

Removes L2 regularization from loss gradient, incorporates it in weight update. Improves convergence over Adam.

1.2.3 Common Loss Functions

Cross-Entropy Loss

  • Binary: BCE = -y log(ŷ) - (1-y)log(1-ŷ)
  • Multi-class: CE = -Σ yi log(ŷi)

KL Divergence

KL(P||Q) = Σ pi log(pi/qi)

MAE (L1 Loss)

MAE = (1/n) Σ |yi - ŷi|

Less sensitive to outliers than MSE.

MSE (L2 Loss)

MSE = (1/n) Σ (yi - ŷi)2

RMSE

RMSE = √MSE

1.2.4 Regularization

Dropout

During training, randomly drop neurons with probability p:

  • Forward pass: Each neuron dropped with probability p
  • Inference: Scale activations by (1-p)

Weight Regularization

TypeFormulaEffect
LASSOL + λ||θ||1Shrinks coefficients to 0
RidgeL + λ||θ||2Reduces magnitude
Elastic NetL + λ((1-ρ)||θ||1+ρ||θ||2)Trade-off

Early Stopping

Stop training when validation loss reaches plateau or increases.

Batch Normalization

Normalizes inputs across a batch:

x̂ = (x - μB)/√(σB2+ε)
BN(x) = γx̂ + β

Where γ, β are learnable parameters.

1.3 Evaluation

1.3.1 Data Splits

Training Set

  • Used for model f to learn patterns from features xtrain with labels ytrain
  • Should be big enough, representative, and high quality

Evaluation (Validation) Set

  • Used to tune hyperparameters
  • Should be representative proxy, independent of training data

Test Set

  • Only used to report final results
  • Check for: dataset contamination, feature leakage, distribution mismatches

1.3.2 Metrics

Confusion Matrix

ŷ = -ŷ = +
y = -TNFP
y = +FNTP

Accuracy

Accuracy = (TN + TP) / (TN + FN + FP + TP)

Precision & Recall

Precision = TP / (TP + FP)
Recall = TP / (TP + FN)

F1 Score

F1 = 2 × (Precision × Recall) / (Precision + Recall)

ROC Curve

Plots TPR vs FPR at different thresholds:

  • TPR (Recall) = TP/(TP+FN)
  • FPR = FP/(FP+TN)

AUC

Area Under ROC Curve, ranges from 0.5 (random) to 1 (perfect).

1.3.3 Bias-Variance Trade-off

Bias

Difference between average prediction and true value. High bias → underfitting.

Variance

How much predictions change with training data. High variance → overfitting.

StateTraining ErrorBiasVariance
UnderfittingHighHighLow
Just RightSlightly < testBalancedBalanced
OverfittingLowLowHigh

Remedies

  • Underfitting: Complexify model, add features, train longer
  • Overfitting: Regularize, get more data
