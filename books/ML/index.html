Part 1

What is Machine Learning?

Home › Part 1: Foundations › What is ML?
⚙

Overview

Machine learning is a subset of artificial intelligence that enables systems to learn and improve from experience without being explicitly programmed. Instead of following rigid instructions, ML algorithms identify patterns in data and make decisions with minimal human intervention.

The ML Revolution

Traditional programming requires developers to write explicit rules that tell computers how to solve problems. Machine learning takes a different approach: instead of programming the solution, we program the system to learn the solution from examples.

Traditional Programming

Human writes explicit rules (code) that are applied to data to produce answers.

Input: Data + Rules
Process: Computer executes rules
Output: Answers

Machine Learning

System learns rules automatically by finding patterns in data.

Input: Data + Answers
Process: ML algorithm learns rules
Output: Learned rules (model)

Applications of ML

Computer Vision — Image classification, object detection, facial recognition, medical imaging analysis.
Natural Language Processing — Machine translation, sentiment analysis, chatbots, text summarization.
Recommendation Systems — Netflix movie recommendations, Amazon product suggestions, Spotify playlists.
Autonomous Systems — Self-driving cars, robotic control, game-playing agents (AlphaGo, chess AI).
Healthcare — Disease diagnosis, drug discovery, patient outcome prediction, medical imaging.

Types of Machine Learning

Home › Part 1: Foundations › Types of Learning

Supervised Learning

Learning from labeled data where we know the correct answers (ground truth). The algorithm learns to map inputs to outputs.

  • Email spam detection: labeled spam/not-spam emails
  • Image classification: images with category labels
  • Price prediction: historical prices with known values

Unsupervised Learning

Learning from unlabeled data to discover hidden patterns or structures. No ground truth is provided.

  • Customer segmentation: finding natural groupings
  • Anomaly detection: identifying unusual patterns
  • Topic modeling: discovering document themes

Reinforcement Learning

Learning through interaction with an environment, receiving rewards or penalties for actions taken.

  • Game playing: learn optimal strategies
  • Robotics: control policies for manipulation
  • Resource management: optimal allocation

Semi-Supervised and Transfer Learning

Semi-Supervised Learning

Uses a small amount of labeled data combined with a large amount of unlabeled data. This is practical when labeling is expensive but unlabeled data is abundant (e.g., medical imaging, language translation).

Transfer Learning

Leverages knowledge from a related task that has been learned. For example, a model trained to recognize cars can be fine-tuned for truck recognition with less data.

The ML Pipeline

Home › Part 1: Foundations › ML Pipeline

The ML Workflow

1. Problem Definition — Define what problem we're solving, what kind of prediction is needed, and what success looks like.
2. Data Collection — Gather relevant data from databases, APIs, web scraping, or experiments.
3. Data Preparation — Clean data, handle missing values, remove outliers, and transform features.
4. Feature Engineering — Create meaningful features that help the model learn patterns.
5. Model Selection — Choose appropriate algorithms based on problem type and data characteristics.
6. Training — Feed data to the model to learn patterns and optimize parameters.
7. Evaluation — Assess model performance using appropriate metrics on held-out data.
8. Deployment — Deploy the model to production and monitor its performance over time.
9. Iteration — Continuously improve based on feedback and new data.

Supervised Learning

Home › Part 2: Supervised Learning › Regression
📈

Supervised learning uses labeled training data where each example consists of an input and its corresponding desired output. The goal is to learn a mapping function from inputs to outputs that generalizes well to unseen data.

Regression vs Classification

Regression

Predicts continuous numerical values. The output is a quantity.

  • House price prediction
  • Temperature forecasting
  • Stock market prediction

Classification

Predicts discrete class labels. The output is a category.

  • Email spam detection (spam/not spam)
  • Image categorization (cat/dog/bird)
  • Disease diagnosis (positive/negative)

Regression Algorithms

Home › Part 2: Supervised Learning › Regression

Linear Regression

💽

Linear regression finds the best-fitting straight line through the data points. It assumes a linear relationship between input features and the output.

Simple Linear Regression: y = mx + b

where m is the slope and b is the y-intercept.

Multiple Linear Regression: y = β0 + β1x1 + β2x2 + ... + βnxn

Polynomial Regression

When the relationship is nonlinear, polynomial regression fits a curved line to the data:

y = β0 + β1x + β2x^2 + ... + βnx^n

Regularization

Overfitting

High-degree polynomials can fit training data perfectly but fail to generalize to new data. This is called overfitting.

MethodWhat it DoesWhen to Use

Classification Algorithms

Home › Part 2: Supervised Learning › Classification

Logistic Regression

💲

Despite its name, logistic regression is used for classification. It predicts the probability that an input belongs to a particular class using the sigmoid function.

Sigmoid: σ(z) = 1 / (1 + e-z)

The output is a probability between 0 and 1.

Decision Trees

Decision trees make predictions by learning simple decision rules from data features. They are intuitive and easy to interpret.

How Decision Trees Work

At each node, the tree splits the data based on a feature value that best separates the classes. The goal is to maximize information gain or minimize impurity.

Support Vector Machines (SVM)

📏

SVM finds the optimal hyperplane that maximally separates different classes. The support vectors are the data points closest to the decision boundary.

k-Nearest Neighbors (k-NN)

k-NN classifies a data point by looking at its k closest neighbors and taking a majority vote. It's simple but can be slow for large datasets.

Unsupervised Learning

Home › Part 3: Unsupervised Learning › Clustering
👥

Unsupervised learning discovers hidden patterns in data without pre-existing labels. The system explores the structure of the data to find meaningful groups or representations.

Clustering

Clustering groups similar data points together. The number of clusters may be known or discovered.

k-Means

Partitions data into k clusters by minimizing within-cluster variance. Fast and scalable.

  • Initialize k centroids
  • Assign points to nearest centroid
  • Update centroids
  • Repeat until convergence

Hierarchical Clustering

Creates a hierarchy of clusters using dendrograms. Doesn't require specifying k upfront.

  • Agglomerative (bottom-up)
  • Divisive (top-down)

DBSCAN

Density-based clustering that finds arbitrarily-shaped clusters and identifies outliers.

  • Core points: dense regions
  • Border points: near dense regions
  • Noise points: outliers

Dimensionality Reduction

Home › Part 3: Unsupervised Learning › Dimensionality Reduction

Why Reduce Dimensions?

Curse of Dimensionality — As dimensions increase, data becomes sparse and distances lose meaning.
Computational Efficiency — Fewer features mean faster training and inference.
Visualization — Reduce to 2-3 dimensions for visualization.
Noise Reduction — Remove irrelevant or redundant features.

Principal Component Analysis (PCA)

🔬

PCA finds the directions of maximum variance in high-dimensional data and projects it onto a lower-dimensional subspace while preserving as much variance as possible.

Steps:

  1. Standardize the data
  2. Compute covariance matrix
  3. Find eigenvectors (principal components)
  4. Project data onto top k components

t-SNE

t-SNE is a non-linear technique for dimensionality reduction particularly well-suited for visualization of high-dimensional datasets. It preserves local structure.

Neural Networks

Home › Part 4: Neural Networks › Perceptron
🧠

Neural networks are computing systems inspired by biological neural networks. They consist of interconnected nodes (neurons) organized in layers that learn to recognize patterns from data.

The Perceptron

The perceptron is the simplest neural network: a single neuron that makes decisions by weighing up evidence.

Output: y = 1 if (w·x + b) > 0, else 0

where w is the weight vector, x is the input, and b is the bias.

The Perceptron

Limitations of Perceptrons

XOR Problem

A single perceptron cannot learn the XOR function because the classes are not linearly separable. This limitation led to the development of multi-layer networks.

Minsky and Papert's 1969 book "Perceptrons" demonstrated this and temporarily halted neural network research, leading to the "AI winter."

Training Neural Networks

Home › Part 4: Neural Networks › Training

Backpropagation

🔄

Backpropagation computes gradients of the loss with respect to each weight by applying the chain rule layer by layer, propagating error signals backward through the network.

Gradient Descent

Weight update: w ← w - η · ∂L/∂w

where η is the learning rate and ∂L/∂w is the gradient.

Challenges in Training

Vanishing & Exploding Gradients

When gradients become too small (vanishing) or too large (exploding), learning becomes ineffective. This is especially problematic in deep networks with sigmoid/tanh activations.

Overfitting

Deep networks can memorize training data instead of learning generalizable patterns. Techniques like dropout, regularization, and early stopping help prevent this.

Ensemble Methods

Ensemble methods combine multiple models to improve prediction accuracy and robustness.

Random Forest

Ensemble of decision trees trained on random subsets of data and features. Reduces overfitting and variance.

Prediction = majority vote of all trees

Gradient Boosting

Sequentially trains trees to correct errors of previous trees. Popular implementations: XGBoost, LightGBM, CatBoost.

Fm(x) = Fm-1(x) + η·hm(x)

Cross-Validation

k-Fold Cross-Validation

Split data into k folds. Train on k-1 folds and validate on the remaining fold. Repeat k times and average the results. This gives a more robust performance estimate.

Bias-Variance Tradeoff

Total Error = Bias2 + Variance + Irreducible Error

High Bias: Underfitting, model is too simple

High Variance: Overfitting, model is too complex
